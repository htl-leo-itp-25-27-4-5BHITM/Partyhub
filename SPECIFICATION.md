Summary (from proposal)

The project needs a single, comprehensive specification that documents the system's goals, API contracts, data models, and extension points so contributors can make consistent, well-informed decisions. This specification will reduce onboarding time, prevent conflicting implementations, and make architectural trade-offs explicit.
