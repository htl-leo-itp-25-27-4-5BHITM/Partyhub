║                                    PartyHub REST API Structure                                                    ║
╠══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════╣
║                                                                                                                      ║
║  ┌─────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐  ║
║  │  PARTYRESOURCE                                        /api/parties                                           │  ║
║  ├─────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤  ║
║  │  GET    /api/parties                                                                                             │  ║
║  │          ?q=party                    Search (title/description)                                                 │  ║
║  │          ?theme=techno               Filter by theme                                                           │  ║
║  │          ?date_from=15-04-2025        Start date (dd-mm-YYYY)                                                    │  ║
║  │          ?date_to=20-04-2025          End date (dd-mm-YYYY)                                                     │  ║
║  │          ?sort=asc|desc               Sort (default: by date)                                                    │  ║
║  │          X-User-Id: 123               Auth header                                                               │  ║
║  │  POST   /api/parties                                 Create party                                             │  ║
║  │  GET    /api/parties/{id}                            Get party by ID                                          │  ║
║  │  PUT    /api/parties/{id}                            Update party                                             │  ║
║  │  DELETE /api/parties/{id}                            Delete party                                             │  ║
║  │  POST   /api/parties/{id}/join                       Join party                                                │  ║
║  │  DELETE /api/parties/{id}/join                      Leave party                                               │  ║
║  │  GET    /api/parties/{id}/join/status               Check join status                                         │  ║
║  │  GET    /api/parties/{id}/can-edit                   Check if can edit                                         │  ║
║  │  GET    /api/parties/{id}/locations                  Get attendee locations                                   │  ║
║  │  GET    /api/parties/{id}/media                      Get party media                                          │  ║
║  │  POST   /api/parties/{id}/media                      Upload party media                                       │  ║

║                                                                                                                      ║

║  LEGEND                                                                                                               ║
║  ─────────────────────────────────────────────────────────────────────────────────────────────────────────────────   ║
║  GET    (blue)    = Read/Query                                                                                       ║
║  POST   (green)   = Create                                                                                           ║
║  PUT    (yellow)  = Update                                                                                           ║
║  DELETE (red)     = Delete                                                                                            ║
╠══════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════╣
║  API CONVENTIONS                                                                                                     ║
║  ─────────────────────────────────────────────────────────────────────────────────────────────────────────────────   ║
║  • Resources nested under parent resources (max 2 levels)                                                              ║
║  • kebab-case for paths: /api/user-locations → /api/users/{id}/location                                               ║
║  • Direct field parameters for filtering (no filter=type pattern)                                                     ║
║  • Date format: dd-mm-YYYY or dd.MM.yyyy HH:mm                                                                       ║
║  • Sort: ?sort=asc|desc                                                                                              ║
║  • Search: ?q=query                                                                                                  ║
